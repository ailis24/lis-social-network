Вот точный и структурированный промт для Replit Agent. Он написан так, чтобы агент понял архитектуру, не сломал текущий код и реализовал ровно то, что ты просишь.

---

## 📝 **ПРОМТ ДЛЯ AGENT В REPLIТ:**

```javascript
 **ГЛАВНАЯ ЗАДАЧА:**
Добавь в приложение СИСТЕМУ АДМИНИСТРАТОРА с ручным управлением премиумом, проверкой чеков и расширенными правами. 
⚠️ ВАЖНО: НЕ МЕНЯЙ текущую логику приложения (лента, чаты, лисёнок, звонки, анти-залипание). Только ДОБАВЬ новые функции и условный рендеринг для админа.

---

🔐 **1. ОПРЕДЕЛЕНИЕ АДМИНА:**
- Администратором считается ТОЛЬКО аккаунт с номером телефона: `+79999099549`
- При загрузке профиля/сессии проверяй номер телефона пользователя.
- Если номер совпадает → устанавливай флаг `isAdmin: true` в состоянии и БД.
- Все админские элементы интерфейса рендерятся ТОЛЬКО если `isAdmin === true`.
- ⚠️ ВАЖНО: Все действия админа должны проверяться на бэкенде/Firebase Rules, а не только скрываться в UI.

---

👑 **2. ИНТЕРФЕЙС В ПРОФИЛЯХ (Только для админа):**
Когда админ заходит в профиль ЛЮБОГО пользователя:
1. **Кнопка " Премиум"** (справа от ника или в шапке профиля):
   - Если премиум выключен → кнопка серая/контурная "Включить Премиум"
   - Если премиум включен → кнопка золотая "Отключить Премиум"
   - При клике → мгновенно меняет статус в БД и обновляет UI
2. **Отображение номера телефона:**
   - Под никнеймом показывай привязанный номер телефона
   - Обычные пользователи НЕ видят номер (скрывай через условный рендеринг)
   - Админ видит номер у всех пользователей

---

🗑️ **3. УПРАВЛЕНИЕ ПОСТАМИ (Только для админа):**
- В ленте, на стенах профилей, в комментариях:
  - Админ видит иконку 🗑️ "Удалить" на КАЖДОМ посте (не только на своих)
  - При клике → пост удаляется из БД и исчезает из UI у всех
  - Добавь подтверждение: "Удалить пост пользователя [Nickname]?"
- Обычные пользователи видят удаление только для своих постов (как сейчас)

---

🧾 **4. СИСТЕМА ПРОВЕРКИ ЧЕКОВ И ПРЕМИУМА:**
**Логика для пользователя:**
- В разделе "Премиум" добавь кнопку "Загрузить чек об оплате"
- Пользователь вводит номер телефона, с которого платил, и прикрепляет скриншот (jpg/png)
- Чек сохраняется в БД со статусом `status: "pending"`

**Логика для админа:**
- В навигации или профиле админа появляется вкладка " Проверка чеков"
- Там список всех pending-чеков с:
  * Никнеймом пользователя
  * Введенным номером телефона
  * Превью загруженной картинки чека
  * Кнопками: "Написать пользователю", "Одобрить", "Отклонить"
- При клике "Написать пользоват" → открывается личный чат с этим юзером (админ пишет сообщение вручную)
- После проверки админ заходит в профиль пользователя и нажимает 👑 "Включить Премиум"
- При включении премиума:
  * `isPremium: true`
  * `premiumExpiry: Date.now() + 30 дней`
  * В приложении этого пользователя **останавливается шкала анти-залипания** на 30 дней
  * При отключении → `isPremium: false`, шкала продолжает идти

---

 **5. ПЕРЕДАЧА ПРАВ АДМИНА:**
- В настройках аккаунта админа добавь раздел "️ Права администратора"
- Поле ввода: "Введите номер телефона нового админа"
- Кнопка "Передать права"
- Логика:
  1. Проверка, что новый номер существует в БД
  2. У текущего админа `isAdmin: false`
  3. У нового пользователя `isAdmin: true`
  4. Отправка системного уведомления обоим
  5. Перезагрузка сессии/прав в реальном времени

---

💾 **6. ИЗМЕНЕНИЯ В БАЗЕ ДАННЫХ (Firestore/SQLite):**
Обнови схему `users`:
```javascript
{
  phone: string,
  nickname: string,
  isAdmin: boolean, // default: false
  isPremium: boolean, // default: false
  premiumExpiry: timestamp | null,
  // остальные поля без изменений
}
```
Добавь коллекцию `payment_checks`:
```javascript
{
  userId: string,
  userPhone: string,
  imageUrl: string,
  status: "pending" | "approved" | "rejected",
  createdAt: timestamp,
  adminNote: string // опционально
}
```

---

🎨 **7. UI/UX ДЕТАЛИ:**
- Кнопка 👑: размер 40x40px, hover-эффект, плавное переключение цвета
- Номер телефона под ником: мелкий серый шрифт, visible только при `isAdmin`
- Кнопка удаления поста: иконка 🗑️ в правом верхнем углу поста, красная при hover
- Вкладка чеков: таблица/карточки с превью, статусами и действиями
- Все админские элементы должны иметь `z-index` выше обычного контента, но не перекрывать лисёнка

---

⚠️ **8. БЕЗОПАСНОСТЬ И ПРОВЕРКИ:**
- Никогда не доверяй только клиентскому `isAdmin`. Все запросы на удаление постов, выдачу премиума и смену админа должны проходить серверную проверку номера `+79999099549`
- Загруженные чеки сохраняй в защищённую папку `/uploads/checks/` с уникальными именами
- Добавь валидацию: нельзя выдать премиум, если чек ещё в статусе "pending" (админ сначала должен проверить)
- Лог действий админа: записывай в консоль/БД кто, когда и что изменил (выдал премиум, удалил пост, передал права)

---

✅ **9. ПРОВЕРКА ПОСЛЕ РЕАЛИЗАЦИИ:**
1. Зайди под аккаунтом `+79999099549` → убедись, что видишь 👑, номера телефонов, кнопку удаления на чужих постах
2. Зайди под обычным аккаунтом → убедись, что админских элементов НЕТ
3. Загрузи тестовый чек → проверь, что он появился в панели админа
4. Напиши пользователю из панели чеков → сообщение дошло в ЛС
5. Включи премиум через 👑 → шкала анти-залипания остановилась
6. Передай права → старый аккаунт потерял админку, новый получил
7. Удали чужой пост → пост исчез у всех

🚀 **НАЧНИ РЕАЛИЗАЦИЮ. Сохрани весь текущий код без изменений. Добавь только новые компоненты и условный рендеринг.**
```

---

### 💡 Как использовать:
1. Скопируй весь текст между ````javascript` и ````
2. Вставь в чат с Replit Agent
3. Агент начнёт поэтапно внедрять функции, не затрагивая рабочую логику приложения
4. После завершения протестируй по чек-листу в пункте 9

Если агент спросит уточнения по базе данных или путям к файлам — отвечай кратко, ссылаясь на текущую структуру твоего проекта. Всё остальное он сделает сам. 🦊👑
